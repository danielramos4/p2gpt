US02: Login de Client

Com a client enregistrat a l'app imUB

Vull iniciar la sessió del meu compte

Per tal d'accedir a les utilitats de l'aplicació

Acceptance Criteria US 2.1: Login Client: Contrasenya errònia

EN EL CAS QUE la contrasenya introduida per el client sigui errònia

QUAN el client seleccioni accedir a l'aplicació (LOGIN).

EL SISTEMA mostrarà un error "Contrassenya incorrecta".

El sistema dóna opció de login però la contrassenya donada és errònia.

Example del test d'acceptació 2.1
Posem un client existent amb contrassenya errònia: ajaleo@gmail.com amb contrassenya INVENTADA11 el sistema hauria de donar Contrassenya incorrecta.

Acceptance Criteria US 2.2: Login Client: Correu erroni

EN EL CAS QUE el correu no coincideixi amb el de cap compte.

QUAN el client seleccioni accedir a l'aplicació (LOGIN).

EL SISTEMA mostrarà un missatge d'error "Correu inexistent".

El sistema dona opció de login però el correu no coincideix amb el de cap compte.

Example del test d'acceptació 2.2
Posem un client existent amb correu inexistent: gatigos@gmail.com amb contrassenya INVENTADA11 el sistema hauria de donar Correu inexistent.

Acceptance Criteria US 2.3: Login Client: Contrassenya oblidada

EN EL CAS QUE el client introdueixi el seu correu.

QUAN el client seleccioni recuperar contrassenya.

EL SISTEMA retornarà la seva contrassenya.

El sistema retornarà la contrassenya d'un usuari si aquest escriu el seu correu electrònic correctament (això és només un exemple, cibersecurity people dislikes this).

Example del test d'acceptació 2.3
Posem el correu d'un usuari: ajaleo@gmail.com el sistema hauria de donar ajaleoPassw7.

Acceptance Criteria US 2.4: Login Persona: Logins correctes

EN EL CAS QUE el client faci un login correcte.

QUAN seleccioni recuperar contrassenya.

EL SISTEMA li mostrarà el missatge "Login correcte".

El sistema dona opció de login i les dades introduïdes són correctes.
